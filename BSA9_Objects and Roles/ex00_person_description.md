# Описание сущности «Person» (Персона)

## 1. Назначение сущности

Класс «Person» является классом для хранения общих данных о физических лицах в системе барбершопа. Назначение класса определяется требованиями из Use Cases:

- **UC01**: Зарегистрировать клиента (саморегистрация) - хранение регистрационных данных
- **UC03**: Забронировать слот на обслуживание - идентификация клиента для записи
- **UC04**: Зарегистрировать клиента (менеджер) - централизованное управление персональными данными

## 2. Атрибуты сущности

### 2.1 Основные атрибуты

| Наименование | Ключ | Тип данных | Ограничение длины | Обязательность | Значение по умолчанию | Комментарии |
|-------------|------|------------|-------------------|----------------|---------------------|-------------|
| person_id | PK | INT | - | Обязательно | AUTO_INCREMENT | Первичный ключ, уникальный идентификатор персоны |
| login | - | VARCHAR | 50 | Обязательно | - | Логин для доступа в систему, должен быть уникальным |
| password | - | VARCHAR | 255 | Обязательно | - | Пароль для безопасности |
| name | - | VARCHAR | 100 | Обязательно | - | Имя (+отчество) для обращений к персоне |
| phone | - | VARCHAR | 20 | Обязательно | - | Номер телефона |
| registration_date | - | DATETIME | - | Обязательно | CURRENT_TIMESTAMP | Дата и время регистрации в системе |

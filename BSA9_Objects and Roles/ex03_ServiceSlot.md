# Описание сущности "ServiceSlot" (слоты обслуживания)

## 1. Назначение сущности

Сущность содержит рабочие интервалы времени мастеров. Менеджер в системе создает записи, формируя расписание. Клиент выбирает свободные записи объекта для бронирования услуги. Забронированные слоты используются для напоминаний клиентам и формирования отчетов о выполненных услугах. Назначение сущности определяется требованиями из Use Cases: 

- **UC02**: формировать расписание мастера - менеджер создает и управляет свободными слотами.
- **UC03**: Забронировать слот на обслуживание - клиент или система резервирует свободный слот, связывая его с конкретным клиентом и услугой.

## 2. Атрибуты сущности

### 2.1 Основные атрибуты

| Наименование | Ключ | Тип данных | Ограничение длины | Обязательность | Значение по умолчанию | Комментарии |
|-------------|------|------------|-------------------|----------------|---------------------|-------------|
| slot_id | PK | INT | - | Обязательно | AUTO_INCREMENT | Уникальный идентификатор слота (первичный ключ)  |
| date | - | DATE | - | Обязательно | - | Дата слота обслуживания в формате гггг-мм-дд   |
| start_time | - | TIME | - | Обязательно | - | Время начала слота в формате чч:мм:сс  |
| employee_id | FK | INT | - | Обязательно | - | Внешний ключ к сущности "Сотрудник". Идентификатор мастера   |
| client_id | FK | INT | - | Обязательно | - |   Внешний ключ к сущности "Клиенты". NULL = слот свободен  |
| status | - | VARCHAR | 20 | Обязательно | Free | Отслеживает состояние слота в жизненном цикле. Рекомендуемые значения: 'free' (свободен), 'booked' (забронирован), 'completed' (услуга оказана), 'cancelled' (отменен). |
| duration | - | INT | - | Обязательно | 60 | Длительность оказания услуги в минутах |

# Описание сущности «Client» (Клиент)

## 1. Назначение сущности

Сущность "Client"  является сущностью для хранения данных о физических лицах, получающих услуги в системе барбершопа. Назначение сущности определяется требованиями из Use Cases:

- **UC01**: Зарегистрировать клиента (саморегистрация) - хранение персональных данных
- **UC03**: Забронировать слот на обслуживание - идентификация клиента для записи
- **UC04**: Зарегистрировать клиента (менеджер) - управление персональными данными

## 2. Атрибуты сущности

### 2.1 Основные атрибуты

| Наименование | Ключ | Тип данных | Ограничение длины | Обязательность | Значение по умолчанию | Комментарии |
|-------------|------|------------|-------------------|----------------|---------------------|-------------|
| client_id | PK | INT | - | Обязательно | AUTO_INCREMENT | Первичный ключ, уникальный идентификатор записи |
| person_id | FK | INT | - | Обязательно | - | Внешний ключ, уникальный идентификатор персоны |
| preffered_contact_channel | - | VARCHAR | 25 | Обязательно | - | Предпочтительный способ связи с клиентом |
| is_active | - | BOOLEAN | - | Обязательно | FALSE | Факт активности клиента в системе |
| registration_date | - | DATETIME | - | Обязательно | CURRENT_TIMESTAP | Дата и время регистрации в системе |

# Описание сущности «Employee» (Сотрудник)

## 1. Назначение сущности

Сущность "Сотрудник"  является сущностью для хранения данных о физических лицах, работающих в системе барбершопа. Назначение сущности определяется требованиями из Use Cases

- **UC02**: Сформировать расписание мастера - управление данными мастера
- **UC03**: Забронировать слот на обслуживание - управление расписанием мастера 

## 2. Атрибуты сущности

### 2.1 Основные атрибуты

| employee_id | PK | INT | - | Обязательно | AUTO_INCREMENT | Первичный ключ, уникальный идентификатор записи |
| person_id | FK | INT | - | Обязательно | - | Внешний ключ, уникальный идентификатор персоны |
